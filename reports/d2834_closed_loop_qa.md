# D-2834 Closed-Loop QA Reproduction

Generated on 2026-04-22.

## Question

Can the system answer questions by retrieving an HRR fact vector and unbinding
the answer roles, rather than by reading the structured payload directly?

## Protocol

- Store facts as:

```text
fact = bind(SUBJECT, subject) + bind(VERB, verb) + bind(OBJECT, object)
```

- Query with the partial vector:

```text
query = bind(SUBJECT, subject) + bind(VERB, verb)
```

- Retrieve the nearest full fact vector from AMM.
- Unbind the retrieved vector with `VERB` and `OBJECT`.
- Cleanup the unbound vectors against verb and object vocabularies.
- Run 5 continual-learning cycles, one synthetic domain per cycle.
- Evaluate 5 domains x 50 facts x 3 seeds.

## Interpretation

The local implementation reproduces the D-2834 finding: full closed-loop QA is
working in the controlled HRR+AMM setting. The answer is not being recovered
from a prompt or generated by a transformer. The memory retrieves a vector, then
role-unbinding recovers the verb and object slots.

This moves the PoC from memory substrate plus language primitives to a
question-answering loop over persistent HRR facts.
